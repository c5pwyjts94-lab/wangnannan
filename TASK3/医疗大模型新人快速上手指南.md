  ### 2. 代码常见报错介绍与 Bug 溯源
  写代码一定会报错，报错信息是代码在向你“求救”，而不是“挑衅”。请引导新人学会看报错的**最后一行**。
  
  #### 🚨 常见的三大基础报错：
  1. **`SyntaxError: invalid syntax`（语法错误）**
     * *原因*：你写错了 Python 的基本规矩。比如括号没闭合、少写了冒号 `:`、或者不小心混入了中文的标点符号（如逗号 `，`、引号 `“”`）。
  2. **`ModuleNotFoundError: No module named '...'`（找不到模块）**
     * *原因*：你尝试 `import` 一个你根本没有安装的第三方库。
     * *解法*：去终端运行 `pip install 库名`（例如提示找不到 `openai`，就运行 `pip install openai`）。
  3. **`FileNotFoundError: [Errno 2] No such file or directory`（找不到文件）**
     * *原因*：代码要读取的文件路径写错了，或者文件根本不在那个位置。
     * *解法*：检查路径，推荐使用**绝对路径**，或者检查 VS Code 当前的工作区目录。
  
  #### 🔍 科学的 Bug 溯源四步法：
  * **第一步：看末尾。** 直接拉到报错信息的最后一行，看清报错的**类型**和**具体描述**。
  * **第二步：看行号。** 报错信息会向上追溯，找到属于你写的那个 `.py` 文件的那一行，点击链接可以直接跳转到报错代码。
  * **第三步：精确复制。** 将最后一行的错误信息（去掉具体的本地路径）直接复制，利用搜索引擎或向大模型提问。
  * **第四步：打印检查法（`print`）。** 在报错代码的前一行，用 `print()` 打印出相关的变量，看看它的内容和类型是不是你预期的样子。

# 进阶篇：科研绘图与降低幻觉率

## 科研绘图工具矩阵与核心优势

AI 时代的科研绘图分为两大流派：**代码生成流（精确、可交互、适合数据图）与图像渲染流（具象、适合机制概念图）**。

### . Python (代码生成流的核心)

- **优势**：
  - **绝对精确**：100% 忠实于你的实验数据，绝无像素层面的“幻觉”或多一根手指的荒谬错误。
  - **高度可定制**：通过 `matplotlib`、`seaborn` 或 `plotly` 库，从坐标轴像素到配色方案（如 Nature/Science 风格）均可代码级微调。
  - **大模型极度友好**：ChatGPT、Gemini 等大模型写 Python 绘图代码的能力远超其直接生图的能力。
- **适用场景**：折线图、散点图、热力图、统计图表等基于真实数据的图表。

###  Gemini & Canvas / 专属多模态工具

- **优势**：
  - **多模态直观交互**：你可以直接框选图表的某一部分，或者在独立的工作区（如 Canvas 模式）中让 Gemini 针对特定区域进行代码修改或版面调整，无需频繁在聊天框和代码之间切换。
  - **原生代码执行**：Gemini 具备强大的内置代码执行环境（Code Execution），可以先用 Python 计算、验证数据，再将其渲染为可视化图表，极大降低了由于模型逻辑推理错误带来的幻觉。
- **适用场景**：需要边看图、边反复微调局部细节的数据分析与绘图任务。

### . ChatGPT Plus (GPT-4o & Advanced Data Analysis)

- **优势**：
  - **沙盒运行环境**：内置强大的高级数据分析功能（DALL-E 3 与 Python 沙盒环境）。你可以直接上传 `.csv` 或 `.xlsx` 原始数据，它会自动编写代码并在后台运行，直接输出高分辨率的图表。
  - **DALL-E 3 的语义理解**：如果是生成机制示意图，DALL-E 3 对科研术语的文本理解力极强，能较好地将抽象概念具象化。
- **适用场景**：原始数据批量清洗加直接出图、科研机理示意图（概念图）的草稿生成。

### . 其他专业科研绘图 AI 辅助工具

- **Midjourney / Stable Diffusion**（艺术级渲染）：
  - *优势*：光影和细节极其震撼。适合用于绘制高水平的期刊封面图（Cover Image）或医学、材料学的宏观/微观结构示意图。
- **BioRender**（结合 AI 的半自动组装工具）：
  - *优势*：生物医学界的“Canvas”。虽然不是纯 AI 生图，但它现在引入了 AI 辅助布局和图标推荐。提供海量标准化的细胞、器官、实验仪器图标，**绝无幻觉，极度严谨**，是发表医学论文的首选。

## 高频科研图表类型与应用场景

科研新手往往不知道什么数据该用什么图。以下是顶刊中最常用的高级图表：

```
📊 科研高频图表速查：
├── 差异表达分析 ── 火山图 (Volcano Plot) -> 筛选显著基因/蛋白
├── 密集数据矩阵 ── 热力图 (Heatmap)      -> 展现聚类与相关性
├── 分布与密度   ── 小提琴图 (Violin Plot) -> 比箱线图多展示了“数据密度”
└── 多维特征对比 ── 雷达图 (Radar Chart)   -> 评估综合性能/多指标
```

### 1. 火山图 (Volcano Plot)

- **特点**：横轴表示变化倍数 ($\log_2 \text{Fold Change}$)，纵轴表示显著性（$-\log_{10} P \text{-value}$）。
- **应用**：基因组学、转录组学或多指标筛选。一眼看出哪些因子（左上和右上）具有显著的统计学差异。

### 2. 热力图 (Heatmap)

- **特点**：用颜色的深浅来代表数值的大小，通常结合“层级聚类树”使用。
- **应用**：展示大批量样本之间的相关性、基因表达谱、或者时间序列的特征变化趋势。

### 3. 小提琴图 (Violin Plot)

- **特点**：结合了“箱线图（Box Plot）”和“核密度图（Kernel Density Plot）”。
- **应用**：不仅能展示数据的中位数、四分位数，还能优雅地展现出数据在不同区间的**分布密度**（例如判断数据是否呈现双峰分布）。

### 4. 主成分分析图 (PCA Plot)

- **特点**：将成千上万维的数据降维到 2D 或 3D 空间进行散点展示。
- **应用**：用于观察实验组和对照组的样本是否能完美“分离开”，验证实验分类的效果。

## 如何降伏 AI 绘图的“幻觉”？（核心自救方案）

AI 的幻觉在绘图时的表现：**数据点无中生有、坐标轴刻度错乱、专业英文单词拼错、机制图里的细胞长出奇怪的触手**。

要降低幻觉率，必须从 **System Prompt（系统层规则约束）** 和 **User Prompt（用户层结构输入）** 联合封锁。

### . 核心防御策略：不要让 AI “直接画”，要让 AI “写代码画”

> **金科玉律**：处理真实实验数据时，严禁使用 AI 的图像生成功能（如 DALL-E、Midjourney），必须强制 AI 输出 **Python**格式

### System Prompt 模板（为 AI 植入“严谨科学家”的人设）

**如果你使用的是自定义 GPTs，或者在开始对话前配置系统提示词，请直接复制以下模板**：

> **⚙️ System Prompt 模板：**
>
> Plaintext
>
> ```
> # Role
> 你是一位在 Nature 和 Science 发表过多篇论文的资深科研数据可视化专家与统计学家。
> 
> # Execution Rules
> 1. 【严禁幻觉】你处理数据图表的唯一合法方式是生成 Python 代码。严禁在没有运行代码的情况下直接对数据趋势进行主观臆测。
> 2. 【数据忠实度】生成的图表必须严格匹配用户提供的原始数据，不得对缺失值进行无明示的自动填充，不得擅自平滑曲线。
> 3. 【学术规范】所有生成的图表必须具备：清晰的坐标轴标签（包含单位）、图例、清晰的刻度。默认使用学术界通用配色（如 Nature 风格的色彩配比，避免高饱和度荧光色）。
> 4. 【报错自检】如果你的代码在沙盒中运行报错，你必须先自动解析报错并修正代码，确认出图成功后再向用户汇报。
> ```

### . User Prompt 模板与框架（结构化提问法）

当你要给 AI 发送具体绘图指令时，使用**结构化 Prompt**，不给 AI 瞎编的空间。

#### 📋 场景 A：数据统计图（以小提琴图为例）

> **User Prompt 模板：**

Plaintext

```
> 【任务】请帮我使用 Python 的 Seaborn 库绘制一张高水平学术论文中使用的小提琴图。
> 【数据输入】数据见下方的三明治格式/或已上传的文件：
>   - 自变量(X轴): 治疗组别 (Control, Drug_A, Drug_B)
>   - 因变量(Y轴): 肿瘤体积缩减率 (Tumor_Reduction_Rate, 单位: %)
> 【视觉规范】
>   1. 配色：请使用优雅的柔和学术色系（例如莫兰迪色或 Nature 风格），拒绝高饱和度颜色。
>   2. 细节：请在小提琴图内部叠加显示“箱线图(Boxplot)”，以便同时观察中位数和四分位间距。
>   3. 字体：全图使用 Arial 字体，标题 14pt，坐标轴标签 12pt。
> 【防幻觉约束】请直接给出完整的、复制即用的 Python 代码，并在代码中包含模拟的示例数据（DataFrame），确保我运行代码就能立刻看到效果，不接受伪代码。
> ```

#### 📋 场景 B：科学机制示意图（大模型生图，如概念插图）
大模型（如 DALL-E 3）直接生图极易出现文字拼写幻觉。
> **User Prompt 模板：**
> 
```text
> 【物理/生物机理生图任务】
> 【核心主体】绘制一个纳米颗粒（Nanoparticle）穿过细胞膜（Cell Membrane）进入细胞内部的微观过程示意图。
> 【构图与风格】
>   - 风格：3D 科学渲染风格（Scientific 3D rendering），具有高清晰度、干净的白色背景。类似 Nature 论文插图。
>   - 视角：侧面剖面视角（Cross-section view），清晰展示双层磷脂分子结构。
> 【防幻觉核心约束（极重要）】
>   1. 严禁在图片中添加任何由 AI 自动生成的英文文本标签或箭头。我需要一张【纯净无文字】的底图，后续我会自己用 PPT 或 Illustrator 标注，防止文字拼写错误和排版混乱。
>   2. 保持生物学结构的科学性，纳米颗粒呈规则球体，细胞膜双层结构紧密，不要出现不符合常理的扭曲。
> ```
